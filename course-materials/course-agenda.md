# Program szkolenia

## AI dla programistów – od pomysłu do MVP

> **Uwaga:** Zaktualizowany program skupiony na workflow pracy z AI i agentami oraz na jednym narzędziu głównym: Claude Code.

---

## Dzień 1: Fundamenty i misja

*Temat przewodni: The AI-Augmented Developer & "odprawa" przed misją*

### Moduł 1.1: The New Frontier

- Omówienie doświadczeń uczestników, poznajmy się
- Wprowadzenie do rewolucji AI w programowaniu
- Obalanie mitów: czy AI Cię zastąpi, czy wzmocni? ASI czy hype?
- Korzyści vs ryzyka (nadmierne poleganie, halucynacje, prywatność, degradacja umiejętności, zanik "mięśni" programistycznych)
- Czy AI nas przyspiesza, czy spowalnia? (badania, case study, opinie)
- Umiejętności i role w IT w dobie Agentów AI: zarządzanie, delegowanie, feedback

**Cel:** Zrozumieć "dlaczego", zmiana mindsetu na AI-first

---

### Moduł 1.2: Kluczowe koncepcje – Modele, Prompty, Vibe Coding / Engineering

- Porównanie modeli (GPT, Claude, Gemini, GLM, open-source / Ollama)
- Prompt Engineering: tokeny, kontekst, attention, zero/few-shot
- Context Engineering: context rot, zarządzanie kontekstem agenta
- Balans między automatyzacją a kontrolą

**Cel:** Poznać fundamenty korzystania z AI w kodowaniu

---

### Moduł 1.3: Mission Briefing & przygotowanie środowiska

- Omówienie projektu (AI Chat do weryfikacji reklamacji na podstawie zdjęcia)
- Omówienie architektury high-level
- Konfiguracja środowiska (Claude Code CLI i Desktop, dowolne IDE, API keys, GIT)

**Cel:** Gotowe środowisko i ustalony cel projektu

---

### Moduł 1.4: Poznaj swoje AI-Powered Coding Tools

- Claude Code, CLI vs Desktop, Wtyczka IDE, Junie/AI Assistant JetBrains
- Funkcje: autocomplete, chat vs Agent, inline edits, integracja z terminalem
- Omówienie najważniejszych funkcjonalności
- Koszty: subskrypcje, API keys, modele lokalne

**Cel:** Poznanie najważniejszych funkcji narzędzi do programowania z AI

---

## Dzień 2: Od pomysłu do projektu

*Temat przewodni: AI jako Twój Product Manager i UX Designer*

### Moduł 2.1: AI pomaga doprecyzować wymagania projektu

- Generowanie pomysłów, walidacja, dokumentacja
- Agent zadaje nam pytania, pomaga doprecyzować jak ma działać aplikacja

**Cel:** Dokument PRD (product requirements) z user stories, opisem funkcjonalnym

---

### Moduł 2.2: Advanced Research & Planowanie projektu

- Wyszukiwanie i analiza dokumentacji bibliotek i frameworków
- Struktura projektu, danych i flow aplikacji
- Tworzenie diagramów architektury, opisów widoków, wireframes

**Cel:** Wybór tech stacka i boilerplate, ADR (Architecture Decision Record)

---

### Moduł 2.3: Konfiguracja narzędzi i umiejętności agentów

- MCP (Model Context Protocol) – serwery i narzędzia (np. Context7, IntelliJ)
- Skills – umiejętności agentów (np. automatyzacja przeglądarki z Playwright)
- Sub-Agenci – konfiguracja specjalistów (np. Java Dev, FE Dev, E2E tester, QA)
- AGENTS.md / CLAUDE.md – opis sposobu pracy agenta, ograniczeń, zasad

**Cel:** Skonfigurowanie naszych agentów aby mogli pracować wydajnie
